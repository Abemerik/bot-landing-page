Описание
Telegram‑бот, выполняющий функции лендинга и информационного канала. Позволяет:

показывать пользователям ключевые разделы (FAQ, «О проекте», «Контакты»);
рассылать сообщения подписчикам Telegram‑канала;
управлять контентом через админ‑панель.

Основные функции:

Для пользователя:

/start — главное меню с разделами;
просмотр FAQ;
раздел «О проекте»;
раздел «Контакты»;
кнопка со ссылкой на канал.

Для администратора:

/admin — вход в админ‑панель;
редактирование текстов (FAQ, /start, «О проекте», «Контакты»);
создание и отправка рассылок (только подписчикам канала).

Технические требования:

Python 3.9;

библиотеки:

pyTelegramBotAPI
python-decouple
requests

Установка и запуск(локально):

1.Скачайте все файлы проекта в одну папку:

bot.py
requirements.txt
README.md

2.Установите зависимости через терминал/командную строку:

bash
---------------------------------
pip install -r requirements.txt
---------------------------------

3.Создайте файл .env (откройте любым текстовым редактором, скопируйте пример ниже и замените значения):

env(пример)
-------------------------------------------------------
TOKEN=8566827261:AAGtX6uPhImHxAaSQEdsdLN_vbQkH-lP4t4
ADMIN_ID=1234567890
CHANNEL=https://t.me/your_channel
CHANNEL_ID=@your_channel
-------------------------------------------------------
Пояснения:

TOKEN — токен вашего бота (получается у @BotFather);
ADMIN_ID — ваш Telegram ID (можно узнать у @userinfobot);
CHANNEL — полная ссылка на канал (формат https://t.me/...);
CHANNEL_ID — юзернейм канала с @ или числовой ID.

!Писать как в примере - без пробелов

4.Запустите бота:

bach
--------------
python bot.py
--------------

Что происходит при первом запуске
Автоматически создаются:

Bot.db — база данных SQLite (с таблицами для хранения текстов);
папка logs — для логов;
файл логов в logs с именем YYYY-MM.log (например, 2025-12.log).

!.Важно:

Каждый месяц (1‑го числа в 00:00) создаётся новый файл логов (по шаблону ГГГГ-ММ.log).
Все логи содержат уровень ошибки, время, модуль и сообщение.

!.Структура проекта

bot.py — основной код бота, обработчики команд и логика;
.env — конфиденциальные данные (токен, ID);
requirements.txt — список зависимостей;
README.md — эта инструкция;
Bot.db — база данных (создаётся автоматически);
logs — папка для логов (создаётся автоматически).

Использование:

Для пользователя:

Отправьте /start → откроется главное меню.
Используйте кнопки для перехода к:
FAQ;
«О проекте»;
«Контакты»;
ссылке на канал.

Для администратора:

Отправьте /admin → вход в админ‑панель.
Следуйте подсказкам бота для:
редактирования текстов;
создания рассылки (фото, текст, кнопки);
предпросмотра и отправки.

!.Важно: рассылка доходит только до пользователей, подписанных на указанный канал.

!.Как работает рассылка
Очередь сообщений (queue.Queue):

Все адресаты добавляются в очередь.
Бот отправляет сообщения последовательно, чтобы избежать бана от Telegram.
Throttling (регулировка скорости):
Начальная задержка между сообщениями: 50 мс.
При ошибке 429 (Too Many Requests) задержка удваивается (максимум до 30 секунд).
После стабилизации отправки задержка постепенно снижается.

!.Обработка ошибок:

429 — бот ждёт и повторяет отправку;
blocked/kicked/unauthorized — пользователь помечается как заблокировавший бота;
Другие ошибки — записываются в лог, отправка продолжается.
Отчёт после рассылки:
Администратор получает сообщение с метриками:

✅ Отправлено;

❌ Ошибки;

🚫 Заблокировали бота.

!.Примечания
База данных: SQLite (Bot.db) хранит тексты разделов и состояние рассылок.
Проверка подписки: через bot.get_chat_member() (статус member/administrator/creator).
Логирование: все действия и ошибки фиксируются в logs/ГГГГ-ММ.log.
Безопасность: токен и ID хранятся только в .env.
